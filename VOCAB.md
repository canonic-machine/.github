## Terms

### APPSTORE

The distribution scope for CANONIC via GitHub.

### VALIDATORS

Concrete implementations of MACHINE enforcement that evaluate candidate states against governing CANON. Distributed as GitHub Actions.

### SERVICES

Business operations that produce artifacts under APPSTORE governance. SERVICES **MUST NOT** modify CANON (non-governing).

### APPS

Instantiated products inheriting from SERVICES templates with full CANONIC governance.

### GitHub

The platform providing user management, billing, discovery, CI/CD, and certification for APPSTORE.

### GitHub Actions

GitHub's CI/CD system for VALIDATORS distribution and enforcement.

### GitHub Marketplace

GitHub's billing platform for SERVICES monetization.

### infrastructure

Custom servers, databases, or compute resources outside GitHub.

### zero infrastructure

The principle that APPSTORE operates entirely on GitHub's platform without custom infrastructure.

### products

SERVICES templates that define governed artifact production (PAPER, BOOK, GRANT, etc.).

### distribution

SERVICES for publishing and delivering governed artifacts.

### protection

SERVICES for intellectual property safeguarding (patents, companies).

### economics

SERVICES for token and coin management.

### GitHub IS the App Store

The axiom that GitHub serves as the sole distribution platform for CANONIC.

### validator distribution

The axiom that VALIDATORS are distributed via GitHub Actions.

### services non-governing

The axiom that SERVICES produce artifacts but do not modify CANON.

### apps instantiation

The axiom that APPS inherit from SERVICES templates.

### CANONBASE

The root workspace containing all scopes in the multi-repo stack. APPSTORE IS the CANONBASE.

### CANONBASE aggregate

The axiom that APPSTORE COVERAGE and ROADMAP aggregate across ALL scopes in the workspace.

### scope

A governed directory containing at minimum a triad (CANON.md, VOCAB.md, README.md).

### multi-repo stack

Multiple git repositories composing a single CANONIC governance system.
